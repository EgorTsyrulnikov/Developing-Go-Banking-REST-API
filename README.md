# REST API Банковского Сервиса

Проект представляет собой REST API для банковского сервиса на языке Go. 

## Технологии

*   **Go 1.23+**
*   **PostgreSQL 17** + `pgcrypto`
*   **Gorilla Mux** - маршрутизация
*   **Logrus** - логирование
*   **JWT** - аутентификация
*   **Bcrypt & HMAC & PGP** - защита данных
*   **Etree & Gomail** - интеграции с ЦБ РФ и SMTP

## Внешние библиотеки и их установка

В проекте используются следующие внешние библиотеки:
*   `github.com/gorilla/mux` — маршрутизатор HTTP-запросов.
*   `github.com/golang-jwt/jwt/v5` — генерация и валидация JWT токенов.
*   `golang.org/x/crypto/bcrypt` — хеширование паролей.
*   `github.com/lib/pq` — драйвер для PostgreSQL.
*   `github.com/sirupsen/logrus` — структурированное логирование.
*   `github.com/beevik/etree` — парсинг XML-ответов от API ЦБ РФ.
*   `github.com/go-mail/mail/v2` — отправка email-уведомлений.
*   `github.com/ProtonMail/go-crypto` — PGP шифрование данных банковских карт.

Установка всех зависимостей происходит автоматически с помощью встроенного пакетного менеджера Go:
```bash
go mod tidy
```

## Запуск проекта

1.  **База данных:**
    Запустите PostgreSQL в Docker с помощью `docker-compose`:
    ```bash
    docker-compose up -d
    ```
    База данных будет доступна на порту 5432, а миграции из папки `migrations` выполнятся автоматически при старте.

2.  **Запуск приложения:**
    Установите зависимости и запустите `main.go`:
    ```bash
    go mod tidy
    go run cmd/api/main.go
    ```

## Доступные эндпоинты

### Публичные
*   `POST /register` — Регистрация пользователя (JSON: username, email, password)
*   `POST /login` — Аутентификация пользователя (JSON: username, password)

### Защищенные (требуют заголовок `Authorization: Bearer <jwt-token>`)
*   `POST /accounts` — Создать счет (JSON: currency)
*   `POST /accounts/deposit` — Пополнить счет (JSON: account_id, amount)
*   `POST /transfer` — Перевод между счетами (JSON: from_account_id, to_account_id, amount)
*   `POST /cards` — Выпустить виртуальную карту по алгоритму Луна (JSON: account_id)
*   `GET /accounts/{accountId}/cards` — Получить список карт (с расшифровкой номера)
*   `POST /credits` — Оформить кредит (расчет ставки по ЦБ РФ + аннуитетные платежи) (JSON: amount, term_months)
*   `GET /credits/{creditId}/schedule` — Посмотреть график платежей по кредиту
*   `GET /accounts/{accountId}/analytics` — Статистика доходов и расходов
*   `GET /accounts/{accountId}/predict?days=30` — Прогноз баланса

## Документация по модулям (Структура проекта)

Проект построен по стандартам Clean Architecture (Чистая архитектура) и Standard Go Project Layout.

*   `cmd/api/main.go` — точка входа в приложение. Инициализирует базу данных, сервисы, роутер и запускает HTTP-сервер.
*   `internal/models/` — структуры данных, описывающие сущности предметной области (User, Account, Card, Credit, Transaction).
*   `internal/repositories/` — слой работы с данными (Data Access Layer). Содержит методы для выполнения SQL-запросов к PostgreSQL.
*   `internal/services/` — слой бизнес-логики. Здесь реализованы правила создания счетов, перевода средств, расчета кредитов по аннуитетной схеме и вычисления ставки с учетом данных ЦБ РФ.
*   `internal/handlers/` — слой HTTP-обработчиков. Принимает HTTP-запросы, валидирует JSON, вызывает сервисы и возвращает HTTP-ответы.
*   `internal/middleware/` — промежуточные обработчики, например, `AuthMiddleware` для проверки JWT токенов.
*   `internal/config/` — загрузка конфигурации из переменных окружения.
*   `internal/scheduler/` — фоновый планировщик, который регулярно проверяет наличие просроченных платежей по кредитам.
*   `pkg/crypto/` — модуль криптографии (шифрование и расшифровка номеров карт с помощью PGP).
*   `pkg/cbr/` — модуль интеграции с API Центрального Банка РФ для получения ключевой ставки (XML парсинг).
*   `pkg/smtp/` — модуль для отправки email-сообщений.
*   `pkg/logger/` — настройка логгера (Logrus).
*   `migrations/` — SQL-скрипты для инициализации структуры БД (создание таблиц, индексов, подключение `pgcrypto`).

## Особенности реализации
- Планировщик платежей по кредитам реализован с помощью `time.Ticker` и запускается в фоне при старте приложения, проверяя просроченные платежи.
- PGP ключи генерируются в памяти при старте, так как это упрощает процесс тестирования без необходимости монтировать внешние ключи.
- SMTP настроен в мок-режиме, письма логируются в stdout.
